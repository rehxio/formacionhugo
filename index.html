¿Qué es Hugo?

Es un generador de sitios estáticos escrito en Go.
Está optimizado para velocidad, fácil uso y configurabilidad.
Hugo toma un directorio con contenido y plantillas y los convierte en un sitio web html completo.
Es un proyecto de código abierto con licencia Apache License 2.0

Instalación

Windows:


- Descargar Hugo para instalarlo nos hará falta descomprimirlo y colocarlo en una carpeta que queramos y luego crear la variable de entorno hacia el .exe - Una vez hecho, para comprobar que está funcionando correctamente, hay que abrir un cmd y escribir: 
hugo --version

Linux:


- En Ubuntu se puede acceder a la tienda de software buscar Hugo e instalar. - Para Debian y Ubuntu se puede instalar por terminal ejecutando: 
sudo apt-get install hugo
- Para Arch Linux habría que ejecutar: 
sudo pacman -Syu hugo
- Para Fedora, Red Hat y CentOS, habría que ejecutar: 
sudo dnf install hugo

Mac OS:


Para instalarlo en Mac necesitamos Homebrew que se instala usando el siguiente comando: 
/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"
Cuando esté instalado el Homebrew hay que ejecutar el siguiente comando para que instalar Hugo: 
brew install hugo
Y por último para comprobar que se ha instalado correctamente habría que ejecutar: 
hugo version

Crear el sitio


Necesitamos crear una carpeta para el proyecto, entraremos a ella mediante terminal y ejecutaremos el siguiente comando para crear un sitio que es la estructura básica de una página hecha con Hugo: 
hugo new site misitio
Sustituimos **misitio** por el nombre del sitio que queramos ponerle, Si queremos utilizar la raiz del proyecto como carpeta predeterminada sustituimos **misitio** por "**.**" Quedando una estructura similar a esta:

Para poder instalar un tema, es necesario que tengamos instalado el **git**, cuando lo tengamos instalado accedemos desde la terminal a la carpeta del sitio que hemos creado y lo inicializamos: 
git init
Ahora tendremos que elegir un tema en la página de temas para hugo.

y luego según el tema que hayamos elegido algunos nos pondrán unos pasos a seguir, sólo será seguirlos, y los que no ponga nada, habrá un enlace a su github, podremos hacer un **git clone**, aunque recomiendo utilizar **git submodule add** en su lugar, ya que de ese modo nos enlaza el tema desde su repositorio pero no mete todos los archivos en nuestro proyecto, sólo será una referencia.

Por ejemplo para añadir el tema Ananke, se haría del siguiente modo: 
git submodule add https://github.com/budparr/gohugo-theme-ananke.git themes/ananke
Luego tendremos que editar el archivo config de nuestro sitio para añadir el tema, lo podemos hacer directamente desde terminal usando: 
echo 'theme = "ananke"' >> config.toml

Content


En esta carpeta tendremos archivos en Markdown para definir que carpeta y archivo html se debe abrir al querer ir a una página de nuestro sitio.

Estos archivos llevan la siguiente estructura: 

							---
							layout: contact
							---
sustituyendo _contact_ por el nombre del html al que queremos hacer referencia.

Data


En esta carpeta estará el contenido del texto y contenido de variables, donde podemos tener parametros para controlar si se muestra una cosa u otra.

En este tipo de ficheros podemos crear la estructura que queramos, lo cual es bueno porque podemos llevar una buena organización.

Un ejemplo de un fichero llamado **contact.yml** sería:

							enable: true
							title: Contacto
							phones:
							  - 690328803
							  - 922423513
							places:
							  - name: ¿Oficina principal?
							    lat: 41.895495
								lon: 12.4773
En este caso enable es un parámetro que usaremos como variable para habilitar o deshabilitar la página, en el caso de _phones_: al poner el guión reconoce que tiene un array de elementos.

Layout


En esta carpeta crearemos la estructura de cada página y sus distintas partes que irán en la carpeta partials, gracias a que hugo permite realizar páginas por partes y así poder integrar una misma parte en varias páginas.

También permite que con una sóla página se puedan crear varias sustituyendo contenido.

Por ejemplo, en el fichero contact.html, sería así: 

							{{ if .Site.Data.contact.enable }}
								{{ partial "header.html" . }}
								{{ partial "contact/body.html" .Site.Data.contact }}
								{{ partial "footer.html" . }}
							{{ end }}

  • Partial: indica a hugo que es una parte de un html mayor.
  • El punto al final de la linea, le pasa el contexto al fichero referenciado.

Static


Como su nombre indica, aquí tendremos todos los ficheros estáticos como suelen imagenes, javascript, css... entre otros.


Funcionamiento


Funcionalidades

Una gran ventaja que tiene hugo es que permite simplificar estructura usando bucles para obtener todos los datos de un campo, de este modo no tenemos que conocer la cantidad de elementos que contiene y se puede actualizar en todo momento, o también permite usar condiciones para poder activar o desactivar contenido que no nos interese mostrar.

Hay varias funcionalidades como pueden ser: _if_, _range_, _with_... existen muchos más, pero estos son los más útiles

Este sería un ejemplo utilizando las 3 funcionalidades

							{{ with .Site.Data.contact }}
								{{ if .enable }}
									<ul>
										{{ range .phones }}
											<li><a href="tel:{{ . }}">{{ . }}</a></li>
										{{ end }}
									</ul>
								{{ end }}
							{{ end }}

Condiciones

Normalmente == != ! >= > <= <
En Hugo eq ne not ge gt le lt


Y a la hora de utilizarlo sería así: 

							{{ if not true }} ¡No entra! {{ end }}
							{{ if ne "Pedro" "Paco" }} ¡Entra! {{ end }}
							{{ if gt 10 5 }} ¡No entra! {{ end }}
							{{ if le 5 5 }} ¡Entra! {{ end }}

Variables

Otra de las funcionalidades que ayudan a facilitar la creación de un sitio web con Hugo es el uso de variables, ya que podemos crear una estructura simple y repetirla.

Por ejemplo en el caso de los miembros y los aprendices, realmente es una sola estructura y se usa con diferente contenido, e incluso dentro de la misma página sólo hay una estructura para una carpeta y esta a su vez hace un range por cada información que tenemos en _data_.

Así es como mostramos la descripción de todos los miembros y aprendices, este código se repite por cada archivo _.yml_ que tenemos en _data_. 

							{{ $lang := .Language.Lang }}
							{{ with .Site.Data.member }}
								<div class="info-card">
									<img class="photo" src="{{ .image }}" alt="{{ .name }} photo" />
									<h2 class="name">{{ .name }}</h2>
									{{ with index . $lang }}
										{{ range .description }}
											<p>{{ . }}</p>
										{{ end }}
									{{ end }}
								</div>
							{{ end }}

Configuración

En el fichero de configuración habría que añadir estas variables para que Hugo reconozca la carpeta de cada idioma 

							# ...
							Languages:
							  es: 
							    contentDir: content/es
							  en: 
							    contentDir: content/en
							# ...

Organización

En el _content_ habrán varias carpetas, una por idioma, que apuntarán a una misma página html del modo que hemos visto antes.

Los diferentes idiomas del texto estarán contenidos en _data_, de este modo tenemos un solo fichero con la información tanto en un idioma como en el otro y con un selector podremos cambiar el contenido de uno al otro. 

							enable: true
							icon: fa fa-contact
							es:
							  title: Contacto
							en:
							  title: Contact

Funcionamiento

Teniendo el fichero **contact.yml** mencionado anteriormente, hay varias formas de acceder a esa información, directamente: 

							{{ .Site.Data.contact.enable }}
							{{ .Site.Data.contact.icon }}
							{{ .Site.Data.contact.es.title }}
							{{ .Site.Data.contact.en.title }}
o de esta otra forma, en la que si añadimos un nuevo idioma, no vamos a tener que cambiar esta información: 

							{{ .Site.Data.contact.enable }}
							{{ with index .Site.Data.contact .Lang }}
								{{ .title }}
							{{ end }}
							{{ .Site.Data.contact.icon }}

Para poder desplegarlo

Para desplegarlo en local por lo general se puede usar: 
hugo server -D
o también puedes usar esta si quieres que se ignore la caché: 
hugo server -D --ignoreCache
Esto levantará la página en un servidor local y te pondrá un enlace con la ip y puerto para poder acceder y verlo en vivo según realizas cambios, cuando lo tengas todo listo y se vea como quieres, sólo hay que usar el siguiente comando: 
hugo
esto te generará una carpeta _public_ con todo el sitio preparado, el contenido de esta carpeta sería lo que debes subir a tu servidor y apuntar con tu servidor web.
